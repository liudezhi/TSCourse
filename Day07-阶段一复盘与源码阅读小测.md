# Day 07：阶段一复盘与源码阅读小测

## 今日目标

复盘 JS/TS 基础心智模型，并完成一次小型源码阅读：从 `package.json` 找入口，再读一个入口文件。

## 90 分钟安排

| 时间 | 任务 | 说明 |
| ----- | ------ | -- |
| 15 分钟 | 概念学习 | 回顾 JS/TS/C 差异 |
| 35 分钟 | 代码练习 | 整理前 6 天代码为一个小目录 |
| 25 分钟 | 源码阅读训练 | 做一次入口追踪小测 |
| 15 分钟 | 复盘笔记 | 回答第 1 周问题 |

## 必学知识点

1. TypeScript 类型与 JavaScript 运行时的边界。
2. `package.json` 是源码阅读入口地图。
3. 模块依赖图要从小处画起。

## C 语言类比

C 项目从编译配置、头文件、`main()` 建立入口意识；TS/Node 项目从 `package.json`、`tsconfig`、入口文件建立入口意识。

## 代码练习

把 Day 1-6 的练习整理为：

```text
phase1-review/
  package.json
  src/
    day01.ts
    day02.ts
    day03.ts
    task.ts
    index.ts
```

新增一个 `README.md`，写出每个文件的作用。

## 源码阅读训练

选择一个小型开源 TS/Node 项目或本地安装包，完成：

1. 找 `package.json`。
2. 找 `bin`/`main`/`exports`。
3. 打开入口文件。
4. 写下入口文件导入了哪些模块。

## 当天产出

- 一个阶段一代码目录。
- 一张入口依赖图。
- 第 1 周复盘笔记。

## 第 1 周复盘问题

- 我能不能解释 TypeScript 和 JavaScript 的关系？
- 我能不能看懂真实源码中的 `import/export`？
- 我能不能从 `package.json` 找到入口？
- 我能不能说清楚 `const` 对对象引用的限制？
- 我能不能画出一个两文件项目的模块依赖？

## 常见坑

- 想一次读完整个项目。
- 入口文件打开后立刻陷入所有细节。
- 不写笔记，第二天重新迷路。

## 过关标准

你能独立完成一次 25 分钟入口追踪，并写出“入口文件做了什么”的 5 行说明。

## 有余力再做

找两个项目比较：一个 CLI 项目和一个库项目的入口字段有什么不同。
